# 成份翻譯機

一個可離線使用的前端翻譯工具：

- 可上傳最新成分翻譯對照表（CSV / TSV / XLSX / JSON）。
- 若未再上傳新檔，系統會自動使用「最新版」對照表。
- 輸入成份或分析值後，選擇目標語言，輸出對應翻譯（支援換行、逗號`,`/`，`、`分隔；多種括號型態內符號不會拆開）。
- 支援以中文、英文、日文任一語言詞條作為輸入來源。

## 啟動方式

可使用任一方式：

```bash
python3 server.py
```

或

```bash
python3 -m http.server 8000
```

說明：
- 系統會優先使用瀏覽器內建 XLSX 解析（不依賴後端 API）。
- 若瀏覽器不支援，才會 fallback 到 `/api/parse-xlsx`（此時請使用 `python3 server.py`）。
- 若預覽環境有路徑前綴（例如 `/xxx/yyy`），系統也會自動嘗試多種 API 路徑。

- 翻譯結果會依目標語言自動整合為一串：中文 `、`、英文 `, `、日文 `、`。

## 建議對照表欄位

可直接使用以下欄位名稱（與你提供的表一致）：

- `中文翻譯`
- `英文名稱`
- `日文名稱`

## XLSX 說明

- 可直接上傳 `.xlsx` / `.xls`。
- 會自動讀取第一個工作表作為對照表。
- 第一列需為欄位名稱（例如：`中文翻譯`、`英文名稱`、`日文名稱`）。

## TSV 範例（建議）

```tsv
中文翻譯	英文名稱	日文名稱
火雞	Turkey	七面鳥肉
鮭魚	Salmon	サーモン
粗蛋白（以上）	Crude Protein (min.)	タンパク質（以上）
```

## 常見錯誤排除

- 若看到 `...non-json response...`，表示 fallback API 收到 HTML 而非 JSON。通常是預覽代理路徑或啟動方式造成。
- 建議先直接用 `python3 server.py` 啟動再測試；若是代理預覽，請確認代理有正確轉發 POST 到 `/api/parse-xlsx`。
